🧱 Estructura de la Base de Datos

Ocho tablas fundamentales que orquestan desde el registro del estudiante hasta el match de su carrera.

👥

PERFILES_USUARIO

Identidad del estudiante. Su ciudad, condiciones socioeconómicas (alojadas en JSONB para alta flexibilidad) y detalles demográficos.

UUID PK
📋

CUESTIONARIOS

Catálogo versionado de las pruebas psicológicas y aptitudinales disponibles en la plataforma.

UUID PK
❓

PREGUNTAS

El banco de ítems. Integra opciones múltiples en un campo JSONB, permitiendo UI dinámica sin tablas secundarias infitas.

FK → Cuestionario
📝

RESULTADOS

Inmutables. Guardan la respuesta final del alumno y el diagnóstico vocacional estructurado de la sesión.

FK → Perfil & Cuestionario
🏛️

INSTITUCIONES

Universidades e institutos técnicos avalados por la plataforma. Contiene información vital de contacto.

Catálogo
🎓

PROGRAMAS

La oferta académica real (ej. Ingeniería Civil). Modela costos, modalidades y requisitos técnicos.

FK → Institución
📅

CONVOCATORIAS

Fechas dinámicas de inscripción, cupos y estado, separadas inteligentemente de los Programas estáticos para no duplicar datos.

FK → Programa
🏆

RECOMENDACIONES

El Match. Une los resultados del test con la oferta del programa, dándole al usuario un porcentaje predictivo de éxito.

Las bases de datos no son estáticas. En Brota tomamos decisiones responsables con la información.

📝 Ejemplo de Inserción (INSERT)

Usado para inicializar la información base (**Seeds**) probando la solidez de las llaves foráneas.

INSERT INTO PROGRAMAS (id, institucion_id, nombre, costo_matricula, activo)
VALUES ('e23f5...', 'fa3a7...', 'Ingeniería de Software', 3500000, TRUE);

⚡ Soft Deletes (UPDATE vs DELETE)

En el mundo real **casi nunca usamos DELETE**. Si una carrera se canceló en Brota, aplicamos un UPDATE para conservar registros estadísticos históricos.

UPDATE CONVOCATORIAS 
SET activa = FALSE, informacion_adicional = 'Cancelada'
WHERE id = 'cc9f1f0...';

🔍 10 Consultas Vitales

Aprovechando SQL Avanzado (LIKE, BETWEEN, IN, IS NULL) para crear filtros predictivos, dashboards y motores de match.

Segmenta instituciones exactas a las ciudades que el estudiante elija.

SELECT nombre, tipo, ciudad FROM INSTITUCIONES 
WHERE ciudad IN ('Bogotá', 'Medellín', 'Cali');

Permite al usuario encontrar carreras que contengan fragmentos de sus intereses.

SELECT nombre, modalidad FROM PROGRAMAS 
WHERE nombre LIKE '%Ingeniería%';

Filtra programas cuyos costos de matrícula se ajustan a un rango determinado.

SELECT nombre, costo_matricula FROM PROGRAMAS 
WHERE costo_matricula BETWEEN 1500000 AND 3000000;

Detecta universidades que olvidaron registrar teléfonos para forzar actualización.

SELECT nombre FROM INSTITUCIONES WHERE telefono IS NULL;

Segmentación por edad colegial (15-17) y ciudad para enviar SMS publicitarios a regiones fijas.

SELECT nombre, edad FROM PERFILES_USUARIO 
WHERE edad BETWEEN 15 AND 17 AND ciudad IN ('Bucaramanga', 'Cartagena');

Búsqueda de encuestas de diagnóstico que estén en fase '1.X'.

SELECT nombre, version FROM CUESTIONARIOS 
WHERE version LIKE '1.%' AND descripcion IS NOT NULL;

Filtra instituciones con correos `.edu.co` oficiales que aún no tienen web adjunta reportada.

SELECT nombre, email FROM INSTITUCIONES 
WHERE email LIKE '%@%.edu.co' AND sitio_web IS NULL;

Obtiene todas las fechas de inscripciones a abrir el próximo semestre en particular (H1 2026).

SELECT nombre, fecha_apertura FROM CONVOCATORIAS 
WHERE fecha_apertura BETWEEN '2026-01-01' AND '2026-06-30';

Alerta. Busca carreras "Médicas/Exactas" a las que se olvidó indicar requisitos restrictivos en BD.

SELECT nombre FROM PROGRAMAS 
WHERE requisitos IS NULL AND area_academica IN ('Salud', 'Ciencias Exactas');

Cruza Usuarios y Recomendaciones para revelarMatches por encima del 90% (Excelente compatibilidad).

SELECT p.nombre, p.apellido, r.compatibilidad 
FROM RECOMENDACIONES r
INNER JOIN RESULTADOS rs ON r.resultado_id = rs.id
INNER JOIN PERFILES_USUARIO p ON rs.perfil_usuario_id = p.id
WHERE r.compatibilidad BETWEEN 90.00 AND 100.00;
